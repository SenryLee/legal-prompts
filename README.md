# 法律工作提示词合集

> 面向律师、法务、法官助理、法律研究员等法律从业者的中文 AI 提示词库。  
> **结构原则：诉讼与非诉分开**——先选大类，再选子类。

## 项目简介

本仓库沉淀可直接复制使用的中文法律工作提示词。统一骨架见 [`docs/PROMPT_SPEC.md`](docs/PROMPT_SPEC.md)。

常见结构包括：

* 明确任务目标  
* 设定可信的法律角色（含场景化立场）  
* 说明输入要求  
* 按场景给出分析框架  
* 输出可直接用于诉讼、谈判、审查、调查或决策支持的结构化结果

## 诉讼 / 非诉 如何一刀切

| 你的任务 | 去哪里 |
|----------|--------|
| 起诉、答辩、上诉、再审、仲裁申请/答辩、庭审、保全、执行 | **诉讼/** |
| 劳动仲裁（申请、答辩、举证、调解） | **诉讼/10** |
| 行政复议、行政诉讼、行政应诉 | **诉讼/11** |
| 破产申请、重整计划、预重整、债权人会议 | **诉讼/12** |
| 刑民交叉程序与风险管理 | **诉讼/13** |
| 审合同、改合同、谈条款 | **非诉/01** |
| 书面咨询、风险评估、方案比选 | **非诉/02** |
| 律师函、法律意见书、催款函、对内材料 | **非诉/03** |
| 解除裁员、制度审查、竞业协议审查（尚未进入仲裁） | **非诉/04** |
| 章程、决议、股权激励、股东知情权行使（诉前） | **非诉/05** |
| 尽调、交易文件、对赌回购条款审查 | **非诉/06** |
| 数据/反垄断/出口管制/披露等专项合规 | **非诉/07** |
| 商标专利著作权审查、无效、布局 | **非诉/08** |

**经验法则**：已经要“立案/开庭/裁决/执行/破产受理”，进诉讼；仍在“审查/设计/谈判/合规整改”，进非诉。

## 怎么选 Prompt（全局）

### 1. 先选诉讼还是非诉

见上表。

### 2. 再按材料类型缩小

* 案卷、证据、判决、对方诉讼文书 → 诉讼  
* 合同文本、制度、尽调资料、合规清单 → 非诉  

### 3. 再按输出类型选子目录

* 风险扫描 / 诊断 → 基础审查、文本诊断、咨询答复、尽调  
* 正式文书 → 诉讼/06 或 非诉/03  
* 谈判与修改 → 合同修改与谈判、交易谈判、和解方案  
* 程序动作 → 诉讼/05、09、11  

### 4. 立场与法域

使用前确认：代表谁、要什么输出、最关心什么指标。  
默认中国大陆法律；涉外/港澳台须另行声明（见 PROMPT_SPEC）。

## 使用建议

* 优先打开对应大类下的 `README.md` 做二次导航  
* 调用模型前补齐：立场、核心材料、目标、约束  
* 诉讼、合同、尽调、合规类任务保留时间线、风险矩阵、优先级清单等结构化输出
