# 法律工作提示词合集

> 面向律师、法务、法官助理、法律研究员等法律从业者的中文 AI 提示词库，覆盖诉讼案件处理、合同审查、非诉法律服务、劳动人事、公司治理、交易与尽职调查、企业合规、知识产权等高频场景。

## 项目简介

本仓库用于沉淀可以直接复制使用的中文法律工作提示词。当前提示词整体仍以统一骨架为目标，常见结构包括：

* 明确任务目标

* 设定可信的法律角色

* 说明输入要求

* 按场景给出分析框架

* 输出可直接用于诉讼、谈判、审查、调查或决策支持的结构化结果

## 分类说明

### 诉讼案件处理

覆盖证据与事实分析、诉讼策略规划、案卷阅读、文书生成、庭审应对、程序申请、执行程序、裁判文书分析、案件策略等场景。

### 合同审查

现已按更清晰的认知路径重组为六类：

* 基础审查：主体资格、效力、格式条款、语言规范、履行监控、风险条款替换、版本对比、谈判策略

* 文本问题诊断：风险点识别、条款缺失、表述规范性、权利义务平衡

* 修改与谈判：修改意见生成、谈判要点提炼

* 条款专题：解除终止、违约责任、付款验收、担保、保密与知识产权、争议解决、不可抗力与情势变更

* 合同类型：买卖、借款、租赁、建设工程、劳动合同、技术合同、保理合同、保证合同

* 商业与交易场景：SaaS、经销代理、Term Sheet、物业、仓储物流、广告发布、咨询服务、居间中介、联营合伙、资产转让、知识产权许可、NDA、意向书/备忘录、委托加工、软件开发、数据处理与委托运营、直播电商合作

### 非诉法律服务

覆盖咨询答复、律师函及法律意见书类文书、内部材料、客户汇报材料、检索报告等场景。

### 劳动人事

覆盖专项处理与制度审查两条主线，包括解除、调岗降薪、批量裁员、工伤职业病、劳务派遣、外包、跨境用工、制度合法性审查等场景。

### 公司治理

覆盖内部治理与专项审查，包括决议、章程、股权架构、股权激励、股东协议、董事会治理、担保与关联交易等场景。

### 交易与尽职调查

覆盖法律尽调风险摘要、尽调红旗问题、股权与资产收购协议、过渡期安排、业绩承诺、交易文件谈判要点等场景。

### 企业合规

覆盖数据合规、广告宣传、商业秘密、平台规则、反垄断、反商业贿赂、出口管制、制裁、环保、安全生产、税务、海关、证券、信息披露、关联交易、内幕交易、培训方案等场景。

### 知识产权

覆盖商标、专利、著作权及综合知识产权管理场景。

## 怎么选 Prompt

如果你不确定该从哪里开始，可以按下面这套全局导航规则选择。

### 1. 先按任务目标选一级模块

* 你要处理已经发生或即将发生的纠纷、诉讼、仲裁、执行、程序申请、庭审准备：看 `诉讼案件处理/`

* 你要审合同、改合同、谈条款、看某类交易协议风险：看 `合同审查/`

* 你要回复客户、出律师函、写法律意见书、做内部材料或汇报：看 `非诉法律服务/`

* 你要处理用工、员工关系、劳动仲裁、制度审查、解除裁员、工伤医疗期：看 `劳动人事/`

* 你要处理公司章程、股东会董事会、股权架构、激励、关联交易、治理机制：看 `公司治理/`

* 你要做投资、并购、融资、尽调、交易文件谈判或交割风险控制：看 `交易与尽职调查/`

* 你要排查监管风险、设计整改方案、准备合规培训或专项合规审查：看 `企业合规/`

* 你要处理商标、专利、著作权、许可、侵权、知识产权布局与争议：看 `知识产权/`

### 2. 再按你手上的材料类型缩小范围

* 你手上主要是案卷、证据、判决书、对方材料：优先看 `诉讼案件处理/`

* 你手上主要是合同文本、协议草案、修订版、谈判条款：优先看 `合同审查/`

* 你手上主要是客户问题、会议纪要、业务背景、管理层需求：优先看 `非诉法律服务/`

* 你手上主要是员工资料、规章制度、考勤薪酬、处分材料：优先看 `劳动人事/`

* 你手上主要是章程、决议、股东协议、授权文件：优先看 `公司治理/`

* 你手上主要是尽调清单、交易文件、投资条款、标的公司资料：优先看 `交易与尽职调查/`

* 你手上主要是制度、流程、问询、处罚、培训需求、专项风险清单：优先看 `企业合规/`

* 你手上主要是权属材料、注册信息、授权协议、侵权比对材料：优先看 `知识产权/`

### 3. 再按你希望得到的输出类型选二级目录

* 想先做全局判断、问题扫描、风险识别：优先选“基础审查”“文本问题诊断”“案卷阅读与分析”“咨询答复”这类目录

* 想直接生成正式文书或近似正式文书：优先选“文书生成”这类目录

* 想要谈判口径、修改建议、优先级排序：优先选“修改与谈判”“案件策略”“诉讼策略规划”这类目录

* 想处理程序动作：优先选“程序申请”“执行程序”“庭审应对”这类目录

* 想做专项问题深挖：优先选“条款专题”“专项合规”“专项处理”“专项审查”这类目录

### 4. 如果是纠纷类任务，再按程序阶段选

* 事实还没整理清楚：先看“证据与事实分析”或“案卷阅读与分析”

* 想决定打不打、怎么打、打哪条路径：先看“诉讼策略规划”或“案件策略”

* 已经要出文书：看“文书生成”

* 已经临近开庭或需要程序动作：看“庭审应对”或“程序申请”

* 已经拿到结果，开始考虑回款或执行：看“执行程序”

### 5. 如果是合同类任务，再按问题性质选

* 先判断合同整体有没有基础风险：看 `基础审查/`

* 先看文本本身有没有明显问题：看 `文本问题诊断/`

* 已经知道要改，想直接出修改意见或谈判要点：看 `修改与谈判/`

* 你关注的是某一类共性条款：看 `条款专题/`

* 你关注的是某一种典型合同：看 `合同类型/`

* 你关注的是某个行业、商业模式或交易结构：看 `商业与交易场景/`

### 6. 如果你在两个模块之间犹豫，可以用这个判断

* “这是合同文本里的风险吗？”优先去 `合同审查/`

* “这是交易推进、交割保护或尽调红旗问题吗？”优先去 `交易与尽职调查/`

* “这是已经进入争议处理或诉讼动作了吗？”优先去 `诉讼案件处理/`

* “这是要出给客户、管理层或业务团队的答复或材料吗？”优先去 `非诉法律服务/`

* “这是劳动用工场景中的合同、制度或争议吗？”优先去 `劳动人事/`

* “这是公司权力结构、决策程序或股东董事关系问题吗？”优先去 `公司治理/`

* “这是监管、制度、处罚、专项审查或整改问题吗？”优先去 `企业合规/`

* “这是围绕知识产权权属、授权、侵权和运营的问题吗？”优先去 `知识产权/`

### 7. 最后用立场做最终筛选

同一个 prompt 在不同立场下，输出方向会明显不同。实际使用前，建议至少先确认：

* 你代表谁：原告/被告、甲方/乙方、投资人/融资方、用人单位/劳动者、许可方/被许可方等

* 你当前最想得到什么：结论、风险排序、文书草稿、修改建议、谈判策略、执行路径

* 你最关心什么：胜率、回款、控制权、责任边界、合规安全、交割落地、商业可执行性

## 使用建议

* 优先按一级模块定位业务场景，再按二级目录选择最接近的任务

* 实际调用模型前，建议先补齐立场、核心材料、目标和约束条件

* 诉讼、合同、尽调、合规类任务，优先保留时间线、风险矩阵、优先级清单等结构化输出
