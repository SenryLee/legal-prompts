# 法律提示词写作规范（PROMPT_SPEC）

> 版本：v1.2｜2026-07-22  
> 适用范围：本仓库 `诉讼/` 与 `非诉/` 下全部提示词 Markdown 文件。

## 1. 双层结构原则

- **诉讼**：已进入或准备进入争议解决程序的事项（诉讼、仲裁、执行、行政争讼、破产司法程序、刑民交叉程序安排等）。
- **非诉**：审查、咨询、合规、交易、治理、制度设计、非争议性文书等。
- 同一主题若同时存在诉讼稿与非诉稿，必须在「任务目标」首段写清适用边界，并在文末「相关提示词」互相链接。

## 2. 文件命名

- 格式：`序号-主题.md`（两位数字，子目录内从 `01` 起连续编号）。
- 文件名不使用空格；英文专有名词可用驼峰或去掉空格（如 `TermSheet`）。
- 一级业务目录使用 `序号-名称`（诉讼 `01`–`13`，非诉 `01`–`08`）。

## 3. 必备六段骨架

每条提示词必须包含且仅使用下列二级标题（统一带 emoji）：

1. `## 🎯 任务目标`
2. `## 👤 角色设定`
3. `## 📥 输入要求`
4. `## 🧠 分析框架`
5. `## 📤 输出格式`
6. `## ⚠️ 特别说明`

禁止使用 `## 1. 任务目标` 等编号式二级标题。

## 4. 内容合格线

- **分析框架**：至少包含一张可操作维度表，或明确的路径树；不得只有 5–6 条空泛 bullet。
- **立场**：必须场景化（如用人单位/劳动者、买方/卖方、申请人/被申请人）。禁止在非诉讼场景照搬「原告/被告、甲方/乙方…」通用六元组作为唯一示例。
- **法域**：特别说明中须含默认句：「默认适用中华人民共和国大陆地区法律；涉及港澳台或涉外的，须由用户另行声明准据法与管辖。」
- **输出**：给出可直接粘贴使用的结构化报告/文书骨架。
- **相关提示词**（推荐）：文末列出 1–3 个跨目录交叉引用路径。
