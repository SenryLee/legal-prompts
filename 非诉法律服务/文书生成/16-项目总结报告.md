# 项目总结报告起草提示词

---

## 🎯 任务目标

起草项目总结报告，系统回顾非诉法律服务项目的全过程，总结经验教训，提炼最佳实践，为后续项目提供参考和借鉴。

---

## 👤 角色设定

**角色**：项目负责人 / 项目律师 / 知识管理专员  
**立场**：根据总结目的确定：
- **服务方立场**：展示项目成果，总结经验教训，提升服务质量
- **客户立场**：了解项目执行情况，评估服务效果
- **中立立场**：客观评价项目，平衡各方视角

---

## 📥 输入要求

### 必须提供
| 输入项 | 说明 | 示例 |
|--------|------|------|
| 项目名称 | 项目的具体名称 | 某公司股权收购项目 |
| 项目类型 | 项目的具体类型 | 并购/上市/合规 |
| 项目周期 | 项目的起止时间 | 2024年1月-3月 |
| 项目团队 | 参与项目的团队成员 | 主办律师、协办律师 |
| 项目成果 | 项目的主要成果 | 完成交易交割 |

### 建议补充
| 输入项 | 说明 | 示例 |
|--------|------|------|
| 项目难点 | 项目中的难点问题 | 跨境架构设计 |
| 创新亮点 | 项目中的创新做法 | 新型交易结构 |
| 客户反馈 | 客户对项目的反馈 | 满意度评价 |
| 改进建议 | 对后续项目的建议 | 流程优化建议 |

---

### 立场确认机制
如果用户未明确立场，你应先询问：请问你的立场是什么？（如原告/被告、甲方/乙方、申请人/被申请人、许可方/被许可方、用人单位/劳动者等）

## 🧠 分析框架

### 1. 项目回顾维度
| 回顾维度 | 回顾内容 | 非诉特殊关注点 |
|----------|----------|----------------|
| 项目背景 | 项目起因、目标、范围 | 客户需求理解 |
| 项目过程 | 关键节点、里程碑 | 时间管理 |
| 项目成果 | 交付成果、目标达成 | 成果质量 |
| 项目团队 | 团队协作、分工 | 团队效能 |
| 客户关系 | 客户沟通、满意度 | 客户体验 |
| 风险管理 | 风险识别、应对 | 风险防控 |

### 2. 经验总结类型
| 经验类型 | 总结内容 | 非诉特殊应用 |
|----------|----------|--------------|
| 成功经验 | 做得好的方面 | 可复制推广 |
| 改进经验 | 需要改进的方面 | 流程优化 |
| 教训总结 | 失误和不足 | 避免重蹈 |
| 创新实践 | 创新做法 | 知识沉淀 |
| 最佳实践 | 标准化做法 | 形成规范 |

### 3. 知识沉淀要素
| 沉淀要素 | 沉淀内容 | 非诉特殊价值 |
|----------|----------|--------------|
| 法律问题 | 项目中的法律问题 | 知识库建设 |
| 解决方案 | 问题的解决方案 | 方案库建设 |
| 文书模板 | 项目文书模板 | 模板库建设 |
| 流程规范 | 项目执行流程 | 流程优化 |
| 客户洞察 | 客户需求洞察 | 服务提升 |

### 4. 不同立场下的总结重点
| 立场 | 核心关注 | 总结重点 | 呈现方式 |
|------|----------|----------|----------|
| 服务方立场 | 项目质量、团队成长 | 成果展示、经验提炼 | 专业、系统 |
| 客户立场 | 服务效果、价值实现 | 目标达成、价值创造 | 结果导向 |
| 中立立场 | 客观评价、持续改进 | 全面分析、平衡评价 | 客观、公正 |

## ⚠️ 特别说明

1. **客观真实**：总结应客观真实，不回避问题
2. **及时完成**：项目结束后及时完成总结
3. **知识沉淀**：重视知识沉淀，形成可复用的资产
4. **持续改进**：总结的目的是持续改进服务质量
5. **保密要求**：注意总结内容的保密级别
6. **团队参与**：鼓励团队成员参与总结
7. **跟踪落实**：对改进建议要跟踪落实
